# AI Prompts Used to Build This Project

This document catalogs every major prompt used with Google Antigravity IDE (Gemini Pro) and OpenCode AI agents to build the Project Management Dashboard. Each phase was driven by an implementation plan, a walkthrough, and a `/startcycle` command — all generated by or in collaboration with AI agents.

The quality of the prompt directly determines the quality of the code. These prompts were crafted carefully to ensure senior-level output at every stage.

---

## agents.md — The Agent Team

The file [`agents.md`](./agents.md) defines the entire AI agent team used throughout the project:

- **Product Manager**: Writes technical specifications, defines pages, components, features, data models
- **UI/UX Designer**: Defines the complete design system — colors, typography, spacing, motion, shadows
- **Frontend Engineer**: Builds all components using React + Vite + TypeScript, following strict patterns
- **Test Engineer**: Writes Vitest unit tests, React Testing Library UI tests, and Playwright E2E tests
- **QA Engineer**: Verifies every page loads correctly, checks responsiveness, animations, forms, theme toggle
- **DevOps Agent**: Scaffolds the project, installs dependencies, configures build tools and test runners

The `agents.md` file also enforces strict rules: TypeScript interfaces with `I` prefix, named exports only, barrel `index.ts` files, `@/` path aliases, no inline destructuring at declaration, Redux for client state only, TanStack Query for server state only, one component per file, co-located test files, and a rigid build verification order (tsc → build → test → coverage → QA).

## phases.md — Phase Breakdown

The file [`phases.md`](./phases.md) defines the 8-phase project structure with status tracking. Each phase depends on the previous one and must pass all verification checks before the next phase begins. Each phase has Frontend Tasks, Test Tasks, and Verification checklists.

## OpenCode AI Performance Optimization Prompts

After the initial phases were completed, targeted performance optimizations were applied using OpenCode AI:

**Sidebar Animation Fix:**
```
In src/components/Sidebar/Sidebar.tsx make sure no Framer Motion width or layout animations are used. Use only CSS transition on className for width change between 64px collapsed and 256px expanded. Remove any Framer Motion animate props that affect width or layout.
Add will-change: transform and transform: translateZ(0) to the sidebar element to force GPU acceleration.
In AppLayout.tsx make sure the main content area uses only a CSS transition for padding-left change and is not re-rendering when sidebar toggles.
```

**Comprehensive Performance Audit:**
```
1. useCallback — wrap any function defined inside a component that is passed as a prop to a child component or used in a useEffect dependency array. Only add useCallback where genuinely needed.
2. React.memo — wrap any component that receives props and does not need to re-render when its parent re-renders for unrelated reasons.
3. useMemo — wrap any expensive computation inside a component that does not need to recompute on every render.
4. Move static arrays, objects, or constants defined inside a component body but never change to outside the component definition.
5. Check all useEffect dependency arrays for missing deps causing stale closures or unnecessary re-runs.
```

**Sidebar Width Adjustment:**
```
In src/components/Sidebar/Sidebar.tsx reduce the expanded sidebar width from 256px to 220px.
Change the corresponding padding-left in AppLayout.tsx from pl-64 to pl-[220px].
Keep collapsed width at 64px unchanged.
```

## How the Prompts Were Structured

Every phase used the same pattern:

1. **Phase context**: Which phase, what it depends on
2. **Agent instructions**: What each agent should build/test
3. **File references**: Specific files to create or modify
4. **Tech constraints**: Which libraries and patterns to use
5. **Rules reminder**: Named exports, interfaces, barrel files, aliases
6. **Verification steps**: The exact commands to run

This structured approach ensured consistent, high-quality output across all phases. The AI agents never deviated from the established patterns because every prompt reinforced them.
